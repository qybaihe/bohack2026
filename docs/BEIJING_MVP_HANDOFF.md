# 《此地有回声：AI 城市时空棋局》北京首发版开发交接文档

版本：V0.2 / React 原型开发交接稿  
范围：北京篇《中轴入局》  
当前目标：先做一条能完整跑通的北京 MVP，后续再叠加天津和正式美术资产。

## 1. 当前决策

第一版采用 React 手机网页 / PWA 形态实现，不使用 Unity。原因是本项目的核心不是重 3D 或战斗系统，而是地图路线、定位、拍照、AI 对话、卡牌收集和游记生成。React 更适合快速迭代、扫码路演、实地测试和后续接入内容 CMS。

推荐技术栈：

- 前端：React + TypeScript + Vite
- UI：自定义 CSS 变量与响应式布局，后续可接 Tailwind
- 图标：lucide-react
- 数据：第一版使用本地 TypeScript 内容配置
- 后端：下一阶段接 Node.js / Fastify 或 Express
- AI：所有模型调用走后端，前端只传地点、照片标签、用户选择、卡牌和任务结果
- 地图：下一阶段接高德地图 JS API 或腾讯地图 JS API
- 打包 App：如需上架或线下展陈，可用 Capacitor 包装 React 项目

## 2. 北京篇产品定位

北京篇名为《中轴入局》。

一句话体验：

用户沿北京中轴线相关公共可达空间行走，通过定位、拍照和时空骰触发历史角色剧场，完成现实观察任务，收集文化卡牌，最终生成一份《我的北京时空游记》。

核心主题：

- 北京讲“秩序与烟火”
- 中轴线是主线，胡同生活是终局回声
- 用户不是听导览的人，而是被城市询问、被角色邀请入局的人

第一版不追求完整北京旅游攻略，只做 5 个高质量棋格。

## 3. 北京主线地图

| 顺序 | 棋格 | 地点 | 核心角色 | 文化卡牌 | 建议体验时间 |
| --- | --- | --- | --- | --- | --- |
| 1 | 城门开市 | 前门 / 正阳门外 | 清末掌柜 / 城门吏 | 商贸卡 / 城门卡 | 10-12 分钟 |
| 2 | 中轴取景 | 中轴视线点 / 天安门外围视角 | 营城匠师 | 秩序卡 | 8-10 分钟 |
| 3 | 宫城水影 | 故宫角楼 / 筒子河外侧 | 宫廷画师 | 宫城卡 | 12-15 分钟 |
| 4 | 登高观城 | 景山 / 景山周边替代点 | 观城史官 | 俯瞰卡 | 15-20 分钟 |
| 5 | 旧城回声 | 鼓楼 / 什刹海 / 胡同入口 | 更夫 / 胡同居民 | 烟火卡 / 回声卡 | 12-15 分钟 |

路线策略：

- 第一版用固定主线，不让骰子决定用户去哪。
- 时空骰只决定本地点的剧情事件类型。
- 若地点不可达，系统允许跳过并触发“错过的来信”。
- 所有地点都要有“不拍照也能继续”的手动兜底。

## 4. 用户入局输入

用户进入北京篇后，需要输入以下内容：

| 输入项 | 选项 | 影响 |
| --- | --- | --- |
| 游玩时间 | 60 分钟 / 90 分钟 / 150 分钟 | 决定主线完整度与支线提示数量 |
| 同行类型 | 独自 / 情侣 / 朋友 / 亲子 / 研学 | 决定任务语气、协作程度和对话难度 |
| 兴趣主题 | 建筑 / 历史人物 / 市井生活 / 摄影 | 决定 AI 追问、照片提示和终局标题倾向 |
| 体力状态 | 轻松走 / 正常走 / 可以多走 | 决定是否提示跳过景山或延伸到什刹海 |

默认推荐组合：

- 时间：90 分钟
- 同行：朋友
- 兴趣：建筑
- 体力：正常走

## 5. 时空骰规则

时空骰有 6 个骰面：

| 骰面 | 作用 | 北京例子 |
| --- | --- | --- |
| 时辰 | 根据时间、天气、光线调整剧情 | 傍晚的角楼会多讲倒影和归鸟 |
| 风物 | 把照片中的元素变成剧情道具 | 屋檐、水面、城门、街声进入台词 |
| 来信 | 角色向用户提出一个问题 | “你们这个时代，还相信方向吗？” |
| 市声 | 触发录音或声音观察任务 | 前门吆喝声、什刹海水声、鼓楼钟声 |
| 转折 | 出现历史处境选择 | 城门关闭、商队到来、宫城召见 |
| 回声 | 用户给城市留下一句话 | 作为游记结尾和海报主文案 |

原型阶段可以用按钮模拟掷骰，也可以随机产生骰面。

## 6. 北京地点内容库

### 6.1 前门 / 正阳门外：城门开市

地点范围：前门大街、正阳门外侧、可安全停留的步行区域。  
用户输入：现场照片；选择“城门 / 招牌 / 人流 / 老字号”。  
AI 判断：GPS 确认地点范围；照片识别门楼、牌匾、街面、人流、光线。  
剧场角色：清末掌柜或城门吏。  
角色开场：

> 你拍到的不是一条街，是城门外最会说话的地方。人从这里进城，货从这里换手，消息也从这里散开。

用户可问：

- 你每天最先看见什么人进城？
- 买卖和城门有什么关系？
- 如果今天的人也从这里经过，你会认出这座城吗？

现实任务：找一个能体现“买卖发生过”的细节，拍招牌、门脸、铺面、排队人群，或录 10 秒街声。  
奖励卡牌：商贸卡；若用户选择城门视角，也可获得城门卡。  
失败兜底：如果照片识别失败，让用户手动选择“我拍到的是招牌 / 街道 / 门楼 / 人流”。

### 6.2 中轴视线点：中轴取景

地点范围：天安门外围或可合法停留的中轴视线点。  
用户输入：拍一张尽量对称的照片；选择“秩序 / 仪式 / 城市尺度”。  
AI 判断：识别道路中线、左右平衡、广场开阔感、天空和人流。  
剧场角色：营城匠师。  
角色开场：

> 你把线拍直了。可真正难的不是画一条线，是让一座城愿意沿着这条线呼吸几百年。

用户可问：

- 为什么一座城需要一条中轴线？
- 方向和权力有什么关系？
- 如果今天重新设计城市，你还会画这条线吗？

现实任务：拍出一张“中轴感”照片，画面中有正中线、左右平衡或远近层次。  
奖励卡牌：秩序卡。  
失败兜底：若现场不便拍照，用户完成观察题：“哪一个元素最能体现中轴秩序？”

### 6.3 故宫角楼 / 筒子河：宫城水影

地点范围：故宫角楼、筒子河外侧、可拍摄角楼与水面的公共区域。  
用户输入：拍摄角楼、屋檐、水面、倒影或天空；选择“建筑 / 权力 / 美”。  
AI 判断：识别屋檐层次、水面倒影、天空颜色、视角远近。  
剧场角色：宫廷画师。  
角色开场：

> 你拍到的不是一座楼，而是一种被安排好的秩序。水替它收住了倒影，屋檐替它收住了天空。

用户可问：

- 你为什么要这样画宫城？
- 空间真的会改变人的心吗？
- 如果你看见今天的游客，会想画下什么？

现实任务：拍一个“角楼与当下同框”的画面：古建筑 + 行人 / 树影 / 天空 / 城市生活。  
奖励卡牌：宫城卡。  
失败兜底：若人流太多或天气差，用户手动选择画面元素，让画师围绕它生成短独白。

### 6.4 景山：登高观城

地点范围：景山公园内可登高观城区域；如无法进入，使用景山周边外侧替代剧情。  
用户输入：拍摄高处城市视野；选择“俯瞰 / 风 / 城市层次 / 远处的线”。  
AI 判断：识别视野开阔度、远近层次、天气和能见度。  
剧场角色：观城史官。  
角色开场：

> 站高一点，城市就不只是街道了。你会看见它把宫城、街巷、树影和人的脚步，都排进同一张长卷。

用户可问：

- 你从高处最先看见什么？
- 这座城的秩序和普通人的生活有什么关系？
- 如果看不见中轴线，它还存在吗？

现实任务：拍一张“城市层次”：近处树木或人，中景建筑，远处天空或轴线。  
奖励卡牌：俯瞰卡。  
失败兜底：如果不能登高，触发“想象登高”剧场：用户选择一个方向，史官讲那个方向连接的城市记忆。

### 6.5 鼓楼 / 什刹海：旧城回声

地点范围：鼓楼外侧、什刹海周边、胡同入口等可停留区域。  
用户输入：拍摄胡同、门牌、湖面、自行车、店铺或街声；留下一句话。  
AI 判断：识别生活元素和声音素材；将宏大中轴叙事转入普通日常。  
剧场角色：更夫或胡同居民。  
角色开场：

> 城楼上的钟声远，胡同里的锅铲近。你走到这里，棋局就不只属于帝王，也属于晚饭前回家的人。

用户可问：

- 你一天里最熟悉的声音是什么？
- 旧城最怕失去什么？
- 我可以把哪句话带到未来？

现实任务：录制 10 秒城市声音，或拍一处“仍在使用的生活细节”。  
奖励卡牌：烟火卡；完成留言后获得回声卡。  
失败兜底：如果用户不想录音，可选择“我听见了车铃 / 人声 / 水声 / 脚步 / 风声”。

## 7. 卡牌系统

| 卡牌 | 获取地点 | 主题 | 影响 |
| --- | --- | --- | --- |
| 城门卡 | 前门 | 边界、进入、规则 | 强化“入城”叙事 |
| 商贸卡 | 前门 | 买卖、街面、人流 | 强化“市井”叙事 |
| 秩序卡 | 中轴视线点 | 对称、方向、规制 | 强化“空间秩序”叙事 |
| 宫城卡 | 故宫角楼 | 建筑、审美、权力空间 | 强化“宫城与构图”叙事 |
| 俯瞰卡 | 景山 | 高处、长卷、城市层次 | 强化“看见整座城”叙事 |
| 烟火卡 | 鼓楼/什刹海 | 胡同、声音、日常 | 强化“生活回声”叙事 |
| 回声卡 | 终局留言 | 用户自己的话 | 决定游记结尾和海报主文案 |

## 8. 终局生成规则

终局生成物包括：

- 个人游记：800-1200 字，原型阶段可先生成短版 300-500 字
- 城市记忆海报：城市、路线、卡牌、用户照片、回声句子
- 卡牌册：显示已获得卡牌和未解锁支线
- 路线回放：按时间线展示用户遇见的角色和完成的任务

标题规则：

- 秩序卡 + 宫城卡 + 俯瞰卡：《一座城如何把时间排成一条线》
- 商贸卡 + 烟火卡 + 回声卡：《我在北京看见秩序旁边的生活》
- 宫城卡 + 烟火卡：《从宫城水影到胡同灯火》
- 城门卡 + 秩序卡：《我从一座城门走进一条轴线》

## 9. 原型开发范围

本轮 React 骨架必须跑通：

1. 城市选择页：先只开放北京，天津显示“下一章”。
2. 入局问答页：收集游玩时间、同行类型、兴趣主题、体力状态。
3. 城市棋盘页：展示 5 个北京棋格和当前进度。
4. 地点页：展示地点信息、拍照上传控件、手动元素选择。
5. 时空骰：模拟随机骰面。
6. AI 剧场页：用本地内容模拟角色开场、用户选择、角色回应。
7. 任务页：完成现实任务并获得卡牌。
8. 卡牌册：展示已获得卡牌。
9. 终局页：根据卡牌生成一个短版北京游记和海报式总结。

暂不做：

- 真实地图 SDK
- 真实定位
- 真实 AI 调用
- 用户登录
- 图片云存储
- 多人协作
- Unity / AR

## 10. 美术资产需求

第一批只为北京生成资产即可：

| 资产 | 数量 | 用途 |
| --- | --- | --- |
| 北京城市封面 | 1 | 城市选择页 |
| 北京地图棋盘背景 | 1 | 城市棋盘页 |
| 5 个地点剧场背景 | 5 | 每个地点的剧场页 |
| 5 个角色半身像 | 5 | AI 剧场角色 |
| 7 张文化卡牌图 | 7 | 卡牌册与奖励弹窗 |
| 6 个时空骰图标 | 6 | 骰面显示 |
| 1 张终局海报底图 | 1 | 游记分享 |
| 3 张状态插画 | 3 | 定位中、照片识别中、生成游记中 |

视觉方向：

- 现代国风 + 城市手账 + 轻剧场感
- 北京色彩：朱红、灰瓦、墨青、暖金、晨昏天空
- 不做写实旅游宣传照，避免和用户现场照片混淆
- 角色像“城市记忆中的人物”，不要过度网游化
