# 《此地有回声》北京横屏棋盘版素材规划与线程交接

版本：V0.1 / 美术资产生产交接稿  
目标：先不处理复杂逻辑，先把 16:9 横屏“大富翁式城市棋盘”页面搭起来，并明确 ImageGen 需要生成的全部图片资产。

## 1. 当前视觉方向

我们已经确定：不要做普通网页仪表盘，也不要做手机端长页面。目标是一个横屏 16:9 的桌面棋盘游戏界面。

核心画面关系：

1. 中央是完整方形棋盘，类似经典棋盘游戏的“外圈走格子”结构，但必须原创，不复制任何具体品牌。
2. 用户棋子沿棋盘格移动。
3. 格子分为：景点格、时空骰格、命运格、拍照任务格、市声任务格、共创格、回声格、终局格。
4. 用户落到景点格后，触发拍照、掷骰、AI 剧场、现实任务、文化卡牌奖励。
5. 右侧面板只展示当前格状态，不抢棋盘主视觉。
6. 底部是玩家已获得卡牌手牌。

视觉气质：

- 现代一点，不要太厚重历史风。
- 保留中国城市文化感，但表达要轻、干净、可玩。
- 像一款完成度较高的桌面数字棋盘游戏，而不是文旅官网。
- 色彩：暖米白、朱红、墨青、灰瓦、暖金、浅青绿、少量墨黑。
- 圆角：8px 左右。
- 资产风格：现代国风 + 城市手账 + 桌游组件感。

## 3. 页面清单与简单设计

### 3.1 入口页

用途：选择城市章节，进入北京《中轴入局》。

需要出现：

- 游戏标题：此地有回声
- 章节卡：北京《中轴入局》

- 开始入局按钮
- 棋盘盒/棋盘预览图
- 少量桌游道具：骰子、卡牌、棋子

页面资产：

- 入口主视觉背景 1 张
- 北京章节封面 1 张
- 天津锁定章节封面 1 张，可稍后做

### 3.2 入局问答页

用途：选择游玩时间、同行方式、兴趣主题、步行状态。

- 规则摘要：固定路线、时空骰事件、拍照触发剧场、收集文化卡
- 生成棋局按钮

页面资产：

- 北京路线预览图 1 张

## 4. ImageGen 素材生产策略

素材不要一张一张全部零散生成。建议分三种方式：

### 4.1 大背景单张生成

适合：

- 页面背景
- 主棋盘底图
- 棋盘中心地图
- 终局故事书
- 入口主视觉

要求：

- 16:9 或方形高分辨率
- 不要把长文字画死在图里，文本尽量用代码层实现
- 图中可以保留无文字装饰框和区域

### 4.2 角色和卡牌单张生成

适合：

- 5 个 AI 角色立绘
- 7 张文化卡牌
- 分享海报模板

要求：

- 角色建议 3:4 或透明背景
- 卡牌建议 2:3 比例
- 卡牌不要把长段文字画死，只画标题位、图像和装饰框

### 4.3 绿幕组件图批量生成

适合：

- 小图标
- 棋子
- 骰子
- 事件 tile 小插画
- 任务图标
- 印章
- 装饰道具

方法：

1. 用 ImageGen 生成一张 sprite sheet，背景为纯绿色 `#00ff00`。
2. 一张图放 12-16 个组件，组件之间留足间距。
3. 生成后用本地脚本抠绿幕，得到透明 PNG。
4. 再切成单个素材。

绿幕 prompt 要强调：

- 背景必须是纯 `#00ff00`
- 不要阴影、渐变、地面、反光
- 组件之间留白
- 不要使用绿色作为组件主体色
- 每个组件居中、完整、边缘清晰

## 6. 代码页面先搭什么

1. 用参考图建立横屏 16:9 `GameShell`。
